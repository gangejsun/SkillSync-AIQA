# PPT Generator

자동으로 전문적인 PowerPoint 프레젠테이션을 생성하는 Claude Code 스킬입니다.

## 특징

- 🎯 **자동 콘텐츠 생성**: 주제만 입력하면 완전한 프레젠테이션 구조 생성
- 🎨 **4가지 컬러 테마**: Blue, Green, Purple, Orange
- 📐 **3가지 디자인 스타일**: Professional, Minimal, Creative
- 📊 **다양한 슬라이드 타입**: 제목, 목차, 본문, 2단 레이아웃, 인용 등
- 🇰🇷 **한글 지원**: 맑은 고딕 폰트로 최적화
- ⚡ **빠른 생성**: 10-15장 슬라이드를 1분 내 생성

## 설치

```bash
# Python 패키지 설치
pip install python-pptx
```

## 사용 방법

### 기본 사용

```bash
# Claude Code에서 다음과 같이 요청하세요:
"AI 기술 트렌드 PPT 만들어줘"
"Next.js 15 소개 발표 자료 12장 만들어줘, 미니멀 스타일로"
"마케팅 전략 프레젠테이션 생성해줘, 초록색 테마로"
```

### 스크립트 직접 사용

```bash
python generate_ppt.py \
  --output presentation.pptx \
  --style professional \
  --color blue << 'EOF'
Slide 1: [제목 슬라이드]
제목: AI 시대의 개발자
부제: 필수 역량과 학습 전략

Slide 2: [목차]
• AI 기술 현황
• 개발자 역할 변화
• 필수 스킬
• 학습 방법

Slide 3: [본문]
제목: AI 기술의 발전
• ChatGPT와 LLM 혁명
• GitHub Copilot 보급
• 생산성 향상 도구들
EOF
```

## 슬라이드 타입

### 1. 제목 슬라이드

```
Slide 1: [제목 슬라이드]
제목: 메인 제목
부제: 서브 타이틀
```

### 2. 목차 슬라이드

```
Slide 2: [목차]
• 항목 1
• 항목 2
• 항목 3
```

### 3. 본문 슬라이드 (불릿 포인트)

```
Slide 3: [본문]
제목: 슬라이드 제목
• 첫 번째 포인트
• 두 번째 포인트
  - 서브 포인트 1
  - 서브 포인트 2
• 세 번째 포인트
```

### 4. 2단 레이아웃 (비교)

```
Slide 4: [2단 레이아웃]
제목: 전통 vs AI 개발

[왼쪽]
• 수동 코딩
• 긴 개발 시간
• 반복 작업

[오른쪽]
• AI 보조
• 빠른 프로토타이핑
• 자동화
```

### 5. 인용 슬라이드

```
Slide 5: [인용]
"AI는 개발자를 대체하지 않는다.
AI를 사용하는 개발자가 그렇지 않은 개발자를 대체한다"
- 샘 알트만
```

## 컬러 테마

### Blue (파랑, 기본)
- 전문적이고 신뢰감 있는 색상
- 비즈니스 프레젠테이션에 적합

### Green (초록)
- 친환경, 성장, 혁신 느낌
- 스타트업, 기술 프레젠테이션에 적합

### Purple (보라)
- 창의적이고 혁신적인 느낌
- 디자인, 크리에이티브 분야에 적합

### Orange (주황)
- 활기차고 열정적인 느낌
- 마케팅, 세일즈 프레젠테이션에 적합

## 디자인 스타일

### Professional (전문적)
- 깔끔한 레이아웃과 명확한 계층
- 비즈니스 환경에 최적화
- 보수적인 컬러 팔레트

### Minimal (미니멀)
- 최대한의 여백 활용
- 슬라이드당 최소한의 텍스트
- 크고 굵은 타이포그래피
- 현대적이고 깔끔한 느낌

### Creative (창의적)
- 비대칭 레이아웃
- 대담한 색상 조합
- 핵심 포인트 시각적 강조
- 피칭과 쇼케이스에 적합

## 콘텐츠 가이드라인

### 좋은 슬라이드 예시 ✅

```
Slide 3: [본문]
제목: GitHub Copilot 활용법

• 코드 자동 완성
  - 컨텍스트 기반 제안
  - 함수 전체 생성

• 주석에서 코드 생성
  - 자연어로 설명
  - 즉시 구현 코드 생성
```

**왜 좋은가?**
- 명확한 제목 (25자 이내)
- 3-5개의 불릿 포인트
- 적절한 계층 구조
- 간결한 텍스트

### 나쁜 슬라이드 예시 ❌

```
Slide 3: 개발자들이 반드시 알아야 할 GitHub Copilot의 모든 기능과 활용 방법
• GitHub Copilot은 OpenAI Codex 모델을 기반으로 하는 AI 페어 프로그래밍 도구로서, Visual Studio Code, JetBrains IDEs, Neovim 등 다양한 에디터에서 사용할 수 있습니다
• 코드 자동 완성 기능은 현재 작성 중인 코드의 컨텍스트를 분석하여 다음에 올 코드를 예측하고 제안합니다
• 주석을 작성하면 해당 주석의 내용을 분석하여 실제 구현 코드를 자동으로 생성해주는 기능도 제공합니다
• 테스트 코드 생성, 리팩토링 제안, 버그 수정 등 다양한 기능을 제공합니다
• 월 10달러의 개인 플랜과 월 19달러의 비즈니스 플랜이 있습니다
```

**왜 나쁜가?**
- 제목이 너무 김 (50자 초과)
- 불릿 포인트가 너무 많음 (5개 이상)
- 각 불릿이 너무 김 (100자 초과)
- 정보 과부하
- 가독성 저하

## 모범 사례

### 1. 구조화
- 도입 → 본론 → 결론 구조 유지
- 슬라이드당 하나의 핵심 메시지
- 논리적 흐름 구성

### 2. 간결성
- 제목: 50자 이내
- 불릿 포인트: 슬라이드당 3-5개
- 각 불릿: 100자 이내
- 계층: 최대 2단계

### 3. 일관성
- 동일한 포맷 유지
- 용어 통일
- 스타일 일관성

### 4. 가독성
- 충분한 여백
- 명확한 폰트 크기
- 적절한 대비
- 시각적 계층

### 5. 스토리텔링
- 명확한 시작과 끝
- 논리적 전개
- 청중 중심 메시지
- 행동 촉구 (Call to Action)

**생성 내용**:
- 제목 슬라이드

## 문제 해결

### python-pptx 설치 오류

```bash
# pip 업그레이드 후 재설치
pip install --upgrade pip
pip install python-pptx
```

### 한글 폰트 깨짐

- Windows: 맑은 고딕 기본 설치
- Mac: 맑은 고딕 설치 필요 (또는 나눔고딕 사용)
- Linux: 한글 폰트 패키지 설치

### 슬라이드 생성 실패

1. 입력 형식 확인 (Slide X: [타입] 형식)
2. 출력 디렉토리 존재 확인
3. 파일 쓰기 권한 확인
4. 다른 프로그램에서 파일 열림 여부 확인
